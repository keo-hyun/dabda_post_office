# Entry Code Mobile Keyboard Design

## Goal
모바일에서 입장 코드 입력 시 가상키보드가 불필요하게 리셋되는 현상을 줄여, 혼합 코드(`DABDA2026`) 입력 경험을 안정화한다.

## Problem
현재 입장 코드 입력은 한 글자 입력할 때마다 앱 상태를 갱신하고, 그 결과 `ENTRY` 화면 전체를 재렌더한다. 이 과정에서 `#entryCode` DOM 노드가 매번 교체되어 모바일 브라우저에서 포커스/키보드 모드가 흔들릴 가능성이 높다.

웹 플랫폼 제약상, 혼합 문자열 입력 필드에서 숫자 키패드를 강제로 고정하는 것은 불가능하다. `inputmode`는 힌트일 뿐이며, 실제 키보드 동작은 OS/브라우저가 결정한다.

## Scope
- 포함: `ENTRY` 화면 입력 안정화
- 포함: 모바일 키보드 개입을 줄이는 입력 속성 정리
- 포함: 회귀 테스트 추가
- 제외: 입장 코드 형식 변경
- 제외: UI를 두 칸 입력 방식으로 바꾸는 구조 변경

## Recommended Approach
1. `ENTRY` 화면에서 `entryCode` 입력 시 전체 `render()`를 호출하지 않고, 입력 요소 값만 상태에 반영한다.
2. 제출/오류/화면 전환 시에만 전체 렌더를 유지한다.
3. 입력 필드에 다음 속성을 추가한다:
   - `autocapitalize="characters"`
   - `autocorrect="off"`
   - `spellcheck="false"`
   - `inputmode="text"`

## Why This Approach
- 웹에서 제어 가능한 부분만 건드린다.
- 현재 디자인/기능 구조를 거의 바꾸지 않는다.
- 이후 디자인 수정 시에도 상태 관리와 DOM 안정성 개선이 독립적으로 유지된다.

## Risks
- 모바일 키보드 모드 지속은 브라우저별로 완전히 같아지지 않을 수 있다.
- 다만 입력 DOM 재생성 제거는 재현된 불편을 줄이는 가장 직접적인 개선이다.

## Verification
- 단위 테스트: 입력 중 `SET_ENTRY_CODE`가 다른 화면 상태를 훼손하지 않는지 확인
- E2E 테스트: 연속 타이핑 후 입력값이 유지되고 포커스가 유지되는지 확인
- 수동 확인: 실제 모바일에서 `DABDA2026` 연속 입력 체감 비교
